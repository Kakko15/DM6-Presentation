Background of the Study

Why electricity forecasting matters for modern energy systems.

Growing Demand

  • Electricity demand rises due to urbanization and technology
  • Developing regions (incl. Philippines) lack monitoring tools
  • Results in energy inefficiencies, losses, and high costs

Time-Series Forecasting

  • ARIMA is widely used for its simplicity and interpretability
  • Effective on univariate data with limited resources
  • Anomaly detection complements forecasting by flagging unusual patterns

Problem Statement

Despite available historical data, many systems fail to use it effectively.

Research Questions

1

How can historical electricity consumption data be used to accurately forecast future usage?

2

How can abnormal or unusual consumption patterns be detected effectively?

3

How can forecasting and anomaly detection be integrated to support energy usage optimization?

Objectives of the Study

General Objective

  • Develop a case study-based system that forecasts electricity consumption and detects anomalies to support energy usage optimization

Specific Objectives

  • Analyze real-world electricity data using time series techniques
  • Implement ARIMA for forecasting future demand
  • Auto-determine optimal ARIMA parameters
  • Evaluate performance using MAE, RMSE, MAPE
  • Detect anomalies using the Z-score method
  • Visualize and interpret results for practical insights

Review of Related Literature

Existing research foundations and identified gap.

Key Studies

  • Box et al. (2015) — ARIMA as standard in time-series
  • Hyndman (2021) — Reliable for short-term forecasting
  • Taylor (2010) — Competes with complex models
  • Chandola et al. (2009) — Residual anomaly detection

Research Gap

  • Studies treat forecasting and anomaly detection separately
  • Few integrate both into a single accessible framework
  • This study bridges the gap with a unified ARIMA + Z-score pipeline

Dataset & Preprocessing

Preparing the data for analysis and modeling.

PJM East Dataset

  • Publicly available electricity load dataset
  • 145,363 records of hourly consumption
  • Measured in megawatts (MW)
  • Date range: 2002 onwards

Preprocessing Steps

  • Convert datetime column to proper format
  • Sort records chronologically
  • Remove duplicate timestamps via aggregation
  • Handle missing values via resampling + averaging
  • Resample to daily, weekly, or monthly
  • 80% training / 20% testing (chronological)

Methodology: ARIMA Model

Autoregressive Integrated Moving Average — the core forecasting engine.

ARIMA(p, d, q)

  • p — Autoregressive order (past values)
  • d — Degree of differencing (stationarity)
  • q — Moving average order (past errors)
  • Assumes linear relationships and stationarity

Automatic Parameter Selection

  • ADF Test — checks stationarity, auto-differencing
  • ACF/PACF — identifies candidate p and q values
  • BIC Grid Search — selects lowest BIC combination
  • Forecasts with 95% confidence intervals

Methodology: Anomaly Detection

Statistical thresholding to flag unusual consumption events.

Z-Score Method

  • Formula: Z = (value − mean) / std
  • Threshold: ±3 standard deviations
  • Values beyond ±3σ flagged as anomalies
  • Visualized as red points on the chart

What Anomalies May Indicate

  • Peak demand periods
  • Equipment malfunction or inefficiencies
  • Energy theft or data errors
  • Abnormal usage behavior

Conclusion & Recommendations

Conclusion

  • ARIMA is effective and interpretable for short-term forecasting
  • Z-score provides simple, practical anomaly detection
  • Integrated system supports energy monitoring and decision-making
  • Deployed as an interactive Streamlit web application

Recommendations

  • Use hybrid models (e.g., ARIMA + LSTM) for improved accuracy
  • Incorporate weather and economic factors
  • Implement real-time monitoring systems
  • Apply advanced anomaly detection (ML-based)
